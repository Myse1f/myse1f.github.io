使用IPFS结合以太坊进行博客系统开发（四）

Author: Myse1f Date: Mar 30, 2020 Updated On: Apr 1, 2020
Tags: IPFS 以太坊

前端框架使用

前端使用的是react框架，以及使用react-router来进行前端路由。这部分知识可以在官方文档上找到学习方法。样式使用的的bootstrap，具体模版为here。

前端使用create-react-app脚手架进行初始化，scss进行样式编辑，可以使用npm install --save node-sass安装编译解析器。只需要在js文件中import相应scss文件即可。

p.s react结合bootstrap写还是有些不方便，推荐使用react的各种样式组件，这里推荐ant-design的组件。

组件

React以组件的方式进行抽象，父组件可以对组件传入属性来控制组件的显示，组件内也拥有自己的state状态，通过内部的状态转换来更改组件的表现形式。

App组件

App组件是顶层组件，应用的整体框架视图。在ComponentDidMount的时候加载博客数据。

子组件有NavBar和Footer，其余组件根据路由信息加载。

Navbar

导航栏组件，设置<Link>路由跳转。分别是Home，About，Posts，New Post。

Header组件

头部组件，可以设置背景图片和标题。

PostPreview组件

文章的预览组件，显示文章的标题、副标题和发布时间。

PostProfile组件

最近文章的预览，由一定数量的PostPreview组成。

Footer组件

底部组件，显示网站的信息，以及博主的社交网络信息。

Home组件

主页信息，由头部组件和PostProfile构成。

About组件

有关博主的信息，包含头部组件。

Posts组件

展示所有文章的预览信息，由Header和一组PostPreiew构成。提供搜索器。

这里自己设置分页器，比较麻烦，建议使用ant-design的分页组件。

Post组件

文章展示，由Header组件和文章内容构成。

PostEditor组件

文章的编辑器，使用braft-editor开源组件构成关键编辑器。

NewPost组件

发布新的文章。

EditPost组件

编辑文章。

路由

使用<BrowserRouter>包装App组件即可对应用进行路由。React-Router的路由方式是根据路径加载相应的组件。

url Component
/ Home
/home Home
/about About
/posts Posts
/post/{postcid} Post
/newPost NewPost
/editPost/{postcid} EditPost

数据流

  • 加载App时，从以太坊合约中获取博客的根hash，然后从ipfs中读取所有博客文章的cid
  • 加载Home时，传入最近5篇文章的cid，获取文章的元信息
  • 加载Post时，从路径中获取对应的cid值，加载相应文章的内容
  • 加载Posts时，传入所有文章的cid值，获取文章的元信息
  • 加载EditPost时，传入相应文章的结构
